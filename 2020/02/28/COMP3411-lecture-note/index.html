Decision tree

决策树：是一种树形结构，每一个内部节点表示在某一属性上的测试，每一条分支代表一个测试输出，每一个叶子节点代表一种类别。决策树采用的是自顶向下的递归方法，其基本思想是以信息熵为度量建立一个信息熵下降最快的树。到叶子节点处的熵值为0，每一个叶子节点中的实例都属于同一类

Attribute value：bool，discrete，continuous

ID3

Training set must be consistent

Information Entropy(信息熵): is a measure of how much information we gain when the class value is revealed to us. An entropy of 1 means that they are randomly distributed

信息熵表示一个事件的不确定性，这个不确定性越大我们越难得知这个事件到底是发生还是不发生。如果一个事件发生的结果有两种，任意一种结果发生的概率超过0.5，这个事件的不确定性就会减小，因为我们知道概率大于0.5的那个结果很可能会发生。一个事件的结果可能有多种，这多种结果发生的概率之间差别越大，这个事件的不确定性越小即它的信息熵越小，因为发生概率大的结果很可能会发生，所以该事件的不确定性越小。如果一个事件的结果只有一种，那信息熵显然为0。

$p_{i}$表示该事件发生的概率。entropy为0表明分类很清晰，没有混乱数据，当entropy为1时是最混乱的
$$
H(<p1..pn>) = \sum_{n=1} -p_{i}log_{2}p_{i}
$$
比如$$H(<0.5, 0.5>) = 1$$

Suppose we have p positive and n negative examples in a node.
$$
H(<\frac{p}{p+n} + \frac{n}{p+n}>)
$$

Let Ei have pi positive and ni negative examples：
$$
H(<\frac{p_{i}}{p_{i}+n_{i}} + \frac{n_{i}}{p_{i}+n_{i}}>)
$$
an then we can get:
$$
\sum_{i}\frac{p_{i}+n_{i}}{p+n}H(<\frac{p_{i}}{p_{i}+n_{i}} + \frac{n_{i}}{p_{i}+n_{i}}>)
$$

Pruning Of Trees

修剪树有可能会提升准确率。

  1. For given tree, estimate its expected error
  2. Also estimate errors for variously pruned tree
  3. Choose tree with minimal estimated error

We can estimate the error rate on the (unseen) test items using the Laplace error:
$$
E=1-\frac{n+1}{N+k}
$$
N = total number of training examples

n = number of training examples in the majority class

k = number of classes

decision tree 是分类算法的一种，分类器预测每个实例的类：如果正确，则视为成功；否则，将视为成功。如果不是，则为错误。
